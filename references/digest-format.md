# 日报格式

Markdown 日报由 `new_only.json`（`scripts/dedupe.py` 的输出）渲染。渲染工作由 `scripts/digest.py render` 完成，SKILL.md 不需要手写格式化逻辑。标题里的季节标签、岗位方向标签都是 `digest.py` 从 `config.json` 的 `target_season_label`/`job_category_label` 读出来拼的，不是写死的字符串——下面模板里的"2026秋招"、"财务"、分数等都只是示例值。

传了 `--resumes`（标准 v2 调用）才渲染匹配区块；传 `--evaluated` 才渲染"本次补评完成"小节。都不传时输出与 v1 完全一致（老用户/无简历场景兼容）。

## Markdown 日报模板（有新增时）

```
# 2026秋招 财务方向岗位监控日报 · 2026-08-01

发现 2 家新开放企业，共 3 个新增岗位（已深评 2，待深评 1）

## 某某银行
- **风控管培生** — 上海
  匹配：84/100 · 推荐投递 · 建议一键定制简历
  优势：Python+SQL 与JD硬性要求完全对口
  差距：JD偏好有金融实习经历，简历暂无
  亮点：CPA优先，2027届
  链接：https://...
  来源：牛客网 · 发现于 2026-08-01
- **总行科技岗** — 北京
  匹配：待深评（下次运行自动补评）
  亮点：...
  链接：...
  来源：...

## 本次补评完成（此前"待深评"岗位）
- 某咨询公司 · 数据分析岗 — 76/100，优势：...

当前已监控 12 家公司 / 27 个历史岗位 · 已深评 8 · 待深评 3。
```

要点：
- 公司分组按"组内最高分"降序排（84 分的公司排最前），无分岗位殿后；组内也按分数降序。
- `建议一键定制简历` 尾缀只在 `score >= resume.min_score_for_action`（默认 70）时出现。
- 抓不到 JD 的岗位标 `匹配：JD未抓到，未评分`。
- 分档含义（常量在 digest.py 顶部，与 references/matching.md 严格一致）：≥85 高度匹配 / 70-84 推荐投递 / 50-69 备选 / <50 观望。

## Markdown 日报模板（无新增时）

```
# 2026秋招 财务方向岗位监控日报 · 2026-08-01

今日未发现新的财务方向正式校招岗位。
当前已监控 12 家公司 / 27 个历史岗位 · 已深评 8 · 待深评 3。
```

这个"无新增"分支必须始终输出，不能空手退出；有补评时"本次补评完成"小节照常附在后面。

## 未导入简历提示行

config `resume.enabled` 为 true 但档案不存在（或无 active_base）时，标题下一行插入（原文由 digest.py 生成）：

```
> 提示：尚未导入简历，本报告未做匹配评分。下次对话时把简历文件发给我即可开启匹配功能。
```

此时不渲染各岗位的匹配行（避免满屏"待深评"与提示行自相矛盾）。
